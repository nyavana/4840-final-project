## ADDED Requirements

### Requirement: Platform device driver with correct compatible string
The kernel module SHALL register as a platform driver with `compatible = "csee4840,pvz_gpu-1.0"`, matching the device tree entry generated by Platform Designer from `pvz_top_hw.tcl`.

#### Scenario: Driver binds to device
- **WHEN** the kernel module is loaded with `insmod pvz_driver.ko` and the device tree contains a matching compatible entry
- **THEN** the driver's probe function SHALL be called and the device SHALL be successfully initialized

#### Scenario: Driver unbind
- **WHEN** the kernel module is removed with `rmmod pvz_driver`
- **THEN** the driver SHALL release all resources cleanly without errors in dmesg

### Requirement: Misc device at /dev/pvz
The driver SHALL register a misc device that creates `/dev/pvz` with standard file operations (open, release, ioctl).

#### Scenario: Device node creation
- **WHEN** the driver probe succeeds
- **THEN** `/dev/pvz` SHALL exist and be accessible from userspace

### Requirement: PVZ_WRITE_BG ioctl
The driver SHALL implement a `PVZ_WRITE_BG` ioctl that accepts a struct containing row (0-3), col (0-7), and color index (0-255), and writes the corresponding value to the BG_CELL register at Avalon byte offset 0x00.

#### Scenario: Write background cell via ioctl
- **WHEN** userspace calls `ioctl(fd, PVZ_WRITE_BG, &bg_arg)` with row=1, col=3, color=2
- **THEN** the driver SHALL compute the packed register value and call `iowrite32` to byte offset 0x00

### Requirement: PVZ_WRITE_SHAPE ioctl
The driver SHALL implement a `PVZ_WRITE_SHAPE` ioctl that accepts a struct containing shape index (0-47), type, visible, x, y, width, height, and color, and writes SHAPE_ADDR, SHAPE_DATA0, SHAPE_DATA1, and SHAPE_COMMIT registers in sequence.

#### Scenario: Write shape entry via ioctl
- **WHEN** userspace calls `ioctl(fd, PVZ_WRITE_SHAPE, &shape_arg)` with valid fields
- **THEN** the driver SHALL write SHAPE_ADDR (offset 0x04), SHAPE_DATA0 (offset 0x08), SHAPE_DATA1 (offset 0x0C), and SHAPE_COMMIT (offset 0x10) using `iowrite32` with byte offsets

### Requirement: PVZ_COMMIT_SHAPES ioctl
The driver SHALL implement a `PVZ_COMMIT_SHAPES` ioctl that writes 1 to the SHAPE_COMMIT register, triggering a commit of the currently addressed shape.

#### Scenario: Commit shapes via ioctl
- **WHEN** userspace calls `ioctl(fd, PVZ_COMMIT_SHAPES, NULL)`
- **THEN** the driver SHALL write 1 to byte offset 0x10

### Requirement: Memory-mapped register access
The driver SHALL use `devm_ioremap_resource` to map the Avalon register region and `iowrite32` for all register writes using byte offsets (base + 4*N).

#### Scenario: Register mapping at probe
- **WHEN** the driver probe function executes
- **THEN** the driver SHALL map the register region from the device tree resource and store the base pointer for ioctl use
